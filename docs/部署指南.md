# 部署指南

## 一、准备工作

### 1. 获取DeepSeek API密钥
1. 访问 https://platform.deepseek.com/
2. 注册账号并登录
3. 进入"API Keys"页面，创建新密钥
4. 复制保存密钥（只显示一次）

### 2. 安装必要软件
- Node.js 18+ (前端)
- Python 3.8+ (后端)
- Git

---

## 二、本地开发环境搭建

### 后端
```bash
# 进入后端目录
cd backend

# 创建虚拟环境（可选但推荐）
python -m venv venv
source venv/bin/activate  # Linux/Mac
# 或 venv\Scripts\activate  # Windows

# 安装依赖
pip install -r requirements.txt

# 创建.env文件
cp .env.example .env
# 编辑.env，填入你的DeepSeek API密钥

# 启动服务
python app.py
```
后端运行在 http://localhost:5000

### 前端
```bash
# 进入前端目录
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```
前端运行在 http://localhost:3000

---

## 三、线上部署

### 方案一：Vercel + Railway（推荐，免费）

#### 1. 部署后端到Railway
1. 访问 https://railway.app/ 并用GitHub登录
2. 点击 "New Project" → "Deploy from GitHub repo"
3. 选择你的仓库，设置：
   - Root Directory: `backend`
   - 添加环境变量：`DEEPSEEK_API_KEY` = 你的密钥
4. 部署成功后获得一个URL，如：`https://xxx.railway.app`

#### 2. 部署前端到Vercel
1. 访问 https://vercel.com/ 并用GitHub登录
2. 点击 "New Project" → Import你的GitHub仓库
3. 设置：
   - Root Directory: `frontend`
   - Framework Preset: Vite
4. 添加环境变量：
   - `VITE_API_BASE` = 你的Railway后端地址
5. 点击Deploy

---

### 方案二：使用云服务器

如果你有云服务器（阿里云/腾讯云等）：

```bash
# 1. 上传代码
scp -r 古籍解读项目 root@your-server-ip:/root/

# 2. SSH登录服务器
ssh root@your-server-ip

# 3. 后端部署
cd /root/古籍解读项目/backend
pip install -r requirements.txt
# 创建.env并填入API密钥
nohup gunicorn -w 4 -b 0.0.0.0:5000 app:app &

# 4. 前端部署
cd /root/古籍解读项目/frontend
npm install
npm run build
# 使用nginx托管dist目录
```

## 四、快速测试清单

部署完成后，按以下步骤测试：

1. ✅ 访问前端网址，页面正常显示
2. ✅ 点击"加载示例"，古文内容正常显示
3. ✅ 点击"翻译"，等待几秒后显示翻译结果
4. ✅ 切换到"深度解读"，输入问题测试AI问答
5. ✅ 在输入框中选中词语，测试划词注释（如果实现）

## 五、常见问题

### Q: 翻译没反应？
A: 检查后端是否正常运行，API密钥是否正确配置

### Q: 跨域错误？
A: 确保后端已启用CORS，或前端配置了正确的API代理

### Q: 部署后API调用失败？
A: 检查Vercel环境变量`VITE_API_BASE`是否正确设置为后端地址
